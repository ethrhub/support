# Contributing to EthrHub

Thank you for your interest in helping improve EthrHub! This repository is for documentation, issue tracking, and community discussions.

## How You Can Help

### Report Issues

Found a bug or problem? Help us fix it!

**Before submitting:**
1. Check the [FAQ](docs/faq.md) - it might already be answered
2. Search [existing issues](https://github.com/ethrhub/hub/issues) to avoid duplicates
3. Try the [troubleshooting guide](docs/troubleshooting.md)

**When reporting:**
- Use our [bug report template](.github/ISSUE_TEMPLATE/bug_report.md)
- Include clear steps to reproduce
- Add screenshots if helpful
- Mention your plan type (Basic/Premium)
- Include browser/OS information

### Request Features

Have an idea for a new feature?

**Before requesting:**
- Check [existing requests](https://github.com/ethrhub/hub/issues?q=is%3Aissue+label%3Aenhancement)
- Consider if it benefits other users
- Think about how it fits with EthrHub's goals

**When requesting:**
- Use our [feature request template](.github/ISSUE_TEMPLATE/feature_request.md)
- Explain the problem it solves
- Describe your proposed solution
- Share who would benefit

### Improve Documentation

Help make our docs better!

**You can:**
- Fix typos or unclear wording
- Add helpful examples
- Clarify confusing sections
- Suggest better organization

**How to contribute:**
1. Click "Edit" on any documentation file
2. Make your changes
3. Submit a pull request
4. We'll review and merge if approved

### Help Others

Join the community and help fellow users!

**Ways to help:**
- Answer questions in [Discussions](https://github.com/ethrhub/hub/discussions)
- Share your tips and best practices
- Help troubleshoot issues
- Welcome new users

## Guidelines

### Issue Reports

- **Be specific** - Include exact error messages and steps
- **Be respectful** - We're all here to help
- **Be patient** - We'll respond as soon as we can
- **Follow up** - Let us know if suggested solutions work

### Feature Requests

- **Explain the why** - Help us understand the use case
- **Be realistic** - Consider technical constraints

- **Stay engaged** - Join discussions about your request

### Documentation

- **Keep it simple** - Write for beginners
- **Test examples** - Make sure commands work
- **Use proper formatting** - Follow markdown conventions
- **Link related docs** - Help users find more info

## Code of Conduct

### Be Respectful

- Treat everyone with respect and kindness
- Welcome diverse perspectives and experiences
- Accept constructive criticism gracefully
- Focus on what's best for the community

### Not Acceptable

- Harassment or discriminatory language
- Personal attacks or insults
- Spam or off-topic content
- Sharing others' private information

**Violations:** Report to support@ethrhub.com. We take this seriously.

## Response Times

### Community Support (Basic Plan)
- Issues: 24-48 hours for initial response
- Discussions: Best effort from community

### Priority Support (Premium Plan)
- Issues: 4-8 hours for initial response
- Email: support@ethrhub.com for faster response
